## üêõ Reporting Bugs

If you find a bug, please help us by reporting it.

1.  **Check Existing Issues**: Before submitting a new issue, please check the existing issues to see if the bug has already been reported.
2.  **Provide Details**: When creating a new issue, please include the following information:
    *   A clear and descriptive title.
    *   The steps to reproduce the bug.
    *   The expected behavior and the actual behavior.
    *   Any relevant error messages or console output.
    *   The environment in which the bug occurred (e.g., Roblox client version, specific executor/environment).

## ‚ú® Suggesting Enhancements

We are always looking for ways to improve Fluent. If you have an idea for a new feature or an enhancement to an existing one, please open an issue with the label `enhancement`.

1.  **Describe the Feature**: Clearly explain the feature and why it would be valuable to the library.
2.  **Provide Examples**: If possible, provide a brief example of how the new feature would be used in code.

## üíª Code Contributions

We appreciate code contributions! To ensure a smooth process, please follow these steps:

1.  **Fork the Repository**: Fork the `Fluent-UI-Library` repository to your own GitHub account.
2.  **Create a Branch**: Create a new branch for your changes. Use a descriptive name like `fix/bug-description` or `feat/new-component`.
3.  **Make Changes**: Implement your changes in your new branch.
4.  **Test**: Ensure your changes do not introduce new bugs and that all existing functionality remains intact.
5.  **Commit**: Write clear, concise commit messages. Follow the convention: `type(scope): subject` (e.g., `fix: Corrected slider value calculation`).
6.  **Pull Request**: Open a Pull Request (PR) against the `main` branch of the original repository.

### Pull Request Guidelines

*   **One Feature/Fix Per PR**: Keep your pull requests focused on a single feature or bug fix.
*   **Clear Description**: Provide a detailed description of your changes in the PR body.
*   **Code Style**: Maintain the existing code style and formatting.

We will review your contribution as quickly as possible and provide feedback. Thank you for helping to make Fluent better!
